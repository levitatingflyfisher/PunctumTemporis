# Contributing to One Second A Day

Thank you for your interest in contributing.

## Reporting Bugs

Open an issue on GitHub with:
- A clear description of the problem
- Steps to reproduce
- Device/OS/Flutter version
- Any relevant logs or screenshots

## Submitting Changes

1. Fork the repository
2. Create a branch from `main` for your change
3. Make your changes, keeping commits focused and descriptive
4. Run `flutter test` and confirm all tests pass
5. Open a pull request against `main` with a description of what changed and why

## Code Style

- Run `dart format .` before submitting
- Follow the existing patterns in the codebase
- Add tests for new behavior where practical
